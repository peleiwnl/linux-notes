A technique used to divide a larger IP network into smaller, more manageable sub-networks (subnets) to improve efficiency and security in IP management. 

## What is subnetting?

A network is divided into smaller subnetworks through the process known as a subnetwork. The network part of an [[IP addressing#IP Address|IP Address]] is created by taking bits from the host party. The host party identifies the specific device on the subnetwork, while the network party identifies the subnetwork as a whole.

The sub-network allows network managers to create more controllable and segmented networks for performance or security needs. For example, a large enterprise could segment its network into subnetworks for multiple divisions or locations.

### Advantages of Subnetting

- Can use the IP address effectively as it divides large networks into smaller ones
- Can introduce additional security to IP addresses
- Can increase performance and reduce network traffic

### Disadvantages of Subnetting

- Can be difficult to change the Subnet structure
- Have to perform good calculations and planning
- If the configuration is not correct, it can bring many external threats
